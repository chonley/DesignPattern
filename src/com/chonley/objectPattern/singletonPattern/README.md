单例设计模式--singleton Design Pattern
----
- 什么是单例设计模式
   - 需要对象唯一，那么消费者在调用时就可以修改同一个变量，但是有
  两种方式进行，一种是用static进行修饰，来保证唯一，但是static的
  生命周期较长，导致内存占用，还有就是创建过多的对象，浪费资源；
  那么这时候，单例设计模式就派上用场了。
  - 场景：配置文件，工具类，线程池，缓存，数据库
  - 步骤：
    1. 私有构造方法
    2. 在本类中new一个对象
    3. 提供一个公共方法返回该对象
  - 不同的写法
     1. 懒汉式
     2. 饿汉式
     3. 双重检验锁
     4. 静态内部类
     5. 枚举 :优点，简单且 thread-safe
